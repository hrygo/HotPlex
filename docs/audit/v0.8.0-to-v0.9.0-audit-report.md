## 1. Security Audit

### 1.1 Critical Findings

#### 🔴 SEC-001: API Key Authentication Disabled by Default (CRITICAL)

| Attribute | Value |
|-----------|-------|
| **File** | `internal/server/security.go` |
| **Lines** | 67-68 |
| **CVSS** | 9.8 (Critical) |
| **Impact** | Unauthenticated remote command execution |

**Description**: Default deployment has NO authentication. Anyone can connect to WebSocket and execute arbitrary shell commands.

```go
// Line 67-68: When no API keys, auth is DISABLED
logger.Info("API key authentication disabled (no keys configured)")
```

**Remediation**:
1. Require API keys by default in production mode
2. Log warning at startup if no keys configured
3. Add environment variable `HOTPLEX_REQUIRE_AUTH=true`

---
